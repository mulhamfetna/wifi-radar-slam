# WiFiSLAM-Sim — datasheet

A **ray-traced** outdoor/vehicular WiFi channel-state-information (CSI) dataset for
SLAM and path-discrimination research. No public outdoor/vehicular WiFi-CSI dataset
exists; this packages the physics-based simulation as an openly-reusable artifact.

> **Important:** this is **simulated** (Sionna RT ray tracing), *not* real hardware
> capture. It is intended for algorithm development and as a reference scenario, and
> to enable supervised path-discrimination (the per-path bounce/interaction labels
> are only available in simulation). Real-hardware validation remains future work.

## Motivation & provenance
- Generated by `experiments/make_dataset.py` from a repo config (e.g.
  `configs/nominal.yaml`) at the release commit. Fully reproducible.
- Scenario: a vehicle drives a straight trajectory through an outdoor scene
  (Sionna built-in `simple_street_canyon_with_cars`) illuminated by ambient WiFi
  access points; the on-vehicle antenna array records CSI per frame.

## Contents (`.npz`, load with `wifi_radar_slam.dataset.CsiDataset.load`)
| key | shape | description |
|-----|-------|-------------|
| `csi` | (n_frames, n_ap, n_rx_ant, n_subcarriers) complex64 | noisy channel frequency response |
| `poses` | (n_frames, 3) | ground-truth vehicle x, y, yaw (m, m, rad) |
| `ap_positions` | (n_ap, 3) | access-point coordinates (m) |
| `gt_map` | (M, 2) | ground-truth facade footprints (m) |
| `paths` | (P, 9) | flat oracle path table (see below) |
| `meta` | JSON | carrier/bandwidth/subcarriers/antennas/scene/SNR/units/license |

### `paths` columns
`[frame, ap, delay_s, phi_r, theta_r, n_bounce, first_interaction_type, object_id, is_floor]`

`n_bounce` (interaction count), `first_interaction_type` (Sionna InteractionType:
0 none, 1 specular, 2 diffuse, 4 refraction, 8 diffraction) and `is_floor` are the
ground-truth **labels** for learned LOS/reflection/floor discrimination.

## Recommended splits & tasks
- **Localization**: use `csi` + `ap_positions`; evaluate against `poses`.
- **Mapping**: reconstruct reflectors; evaluate against `gt_map`.
- **Path discrimination**: classify detections/paths using the `paths` labels.

## Known limitations
Simulated (no hardware impairments beyond AWGN); single straight trajectory per
record; sub-7 GHz unless generated from a 60 GHz config; penetrable default
materials (use a reflective-material config for clean single-bounce returns).
